## Objective

This notebook already contains fully working code that calls the OpenAI Chat Completions API
under different settings — temperature, top_p, max_tokens, stop sequences — and compares two
models on latency and token usage, all on the same Meridian Retail Bank support scenario.

## What each task actually is

1. **Task 1 — Temperature sweep.** Already-written code calls the model 6 times (3 temperatures ×
   2 runs each) on the same banking prompt. Read the code, run it, then run the results cell and
   actually compare the response text across temperatures.
2. **Task 2 — Sampling controls.** Already-written code runs 4 checks: `top_p=0.1`, `top_p=1.0`,
   `max_tokens=20`, and `stop=["."]`. Watch specifically for where `max_tokens` and `stop` cut a
   response off mid-sentence — that's the real-world risk this task is showing you.
3. **Task 3 — Model comparison.** Already-written code calls both models named in the Setup cell
   on the same prompt. Compare the latency and token numbers yourself — don't hard-code a dollar
   price, pricing changes; check OpenAI's current pricing page if you want to reason about cost.
4. **Task 4 — Reflect.** Three short guided questions (1–2 sentences each, not an essay) — answer
   based on what you actually observed in Tasks 1–3, not a generic textbook answer. This is what
   the autograder checks for genuine content, not length.
5. **Task 5 — Stretch (optional, Diamond tier).** Already-written code quantifies reproducibility
   across 5 runs each at temperature 0 and 1.0. Only attempt this if you've finished Tasks 1–4
   with time to spare.
6. Run the final two cells — one prints a comparison table, the other saves
   `m1_1_comparison_results.json` next to the notebook. Without that saved file, the autograder has
   nothing to check.
