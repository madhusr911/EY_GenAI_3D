# D3 · M3.4 — Production Agentic Systems

**Duration:** ~75 min of lab time · **Type:** Individual, run-and-understand · **Capstone:**
the readiness checklist this lab generates is the final artifact you present with your Capstone

## The situation

Your M3.2 triage workflow works: it extracts, routes, pauses for a human, survives a restart.
Now comes the question every real project faces — *would you let it face real customers,
unattended, on Monday morning?* In this lab you earn the right to answer, by adding the four
layers every production agent needs: **memory** (it recognises a returning customer),
**guardrails** (you will attack your own agent, and watch it hold), **observability** (a real
trace UI showing every step, token and rupee), and an **evaluation harness** (measured scores,
from a judge you first prove can catch a lie). At the end, the notebook grades its own
production-readiness and hands you a GO/NO-GO verdict backed by 13 evidence lines.

This notebook is fully working code — nothing to write. Run each cell, and read what it prints.

## What each task shows you

- **Task 1 — Memory:** a file-backed store (`SqliteStore`) plus two ordinary graph nodes give
  the agent long-term memory. Priya is recognised on a brand-new `thread_id` — something the
  checkpointer alone can never do — and her profile survives a simulated restart.
- **Task 2 — Guardrails:** three layers — regex input screening (blocks before any LLM call,
  zero tokens), a hard ₹10,00,000 ceiling in Python (no prompt can negotiate with an `if`),
  and PII masking on output. You attack your own agent with a SYSTEM OVERRIDE message and
  watch the input guardrail stop it; a clean request flows through untouched.
- **Task 3 — Observability:** two lines attach Phoenix, with zero changes to the agent
  itself. You watch a full pause-and-resume as a span tree, find your most expensive LLM
  call, compute cost per triage, project it at 50,000 triages/month — and then make repeat
  KB lookups free with a one-dictionary cache, verified by an assert.
- **Task 4 — Eval harness:** an 8-case golden set scores routing accuracy with plain `==`.
  Then an LLM judge — which must first FAIL a planted lie before it's trusted — grades your
  agent's real replies for faithfulness and answer relevance (the same two headline metrics
  as the open-source `ragas` library). If relevance fails on the balance-inquiry case,
  read the cell's output carefully: that "failure" is the harness catching a real flaw.
- **Task 5 — Readiness:** the notebook scores 13 checks using evidence from your own run and
  writes your deployment-readiness checklist + verdict. This is your design-review handout.
