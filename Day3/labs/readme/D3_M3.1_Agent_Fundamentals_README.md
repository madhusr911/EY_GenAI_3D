# D3 · M3.1 — Agent Fundamentals

**Duration:** ~1.5 hrs · **Type:** Individual, run-and-understand · **Capstone:** first hands-on piece of Phase 3 (wrapping your Day 2 work inside something that can reason about when to use it)

## The situation

Day 1 and Day 2 both had you write, or let LangChain write, a fixed sequence of steps decided in
advance. Today the model decides its own steps. You'll build an agent that chooses between two
tools — a calculator, and a search tool reused from your own Day 2 Meridian article data — and
watch it plan across both when one question genuinely needs them in sequence.

This notebook is fully working code — nothing to write. Run each cell, and read what it prints.

## Work this side by side with the concept page, not start to finish

This module has two files, meant to be used together, not one after the other:

- `Day3/concept/D3_M3.1_Agent_Fundamentals_Concept.html` — five short sections (Foundation, A, B, C, D)
- `Day3/labs/starter/D3_M3.1_Agent_Fundamentals_Starter.ipynb` — the notebook below

The pattern: read a concept section on the page → run the matching task(s) in the notebook →
come back to the page for the next concept → and so on. The notebook's own markdown cells tell
you exactly when to switch back to the page. Five round trips, not one long read followed by one
long lab.

## How to work through this module

1. Open `Day3/concept/D3_M3.1_Agent_Fundamentals_Concept.html` in your browser and
   `Day3/labs/starter/D3_M3.1_Agent_Fundamentals_Starter.ipynb` in Jupyter or VS Code — side by
   side if your screen allows it.
2. Run the notebook's **Setup** cell first — it loads your API key and the same 40 Meridian
   articles from Day 2's M2.3/M2.4 lab.
3. **Foundation + Concept A → Task 1.** Read "What's An Agent, Really?" and Concept A (the ReAct
   step-through), then run Task 1's cell: a single-tool agent, with the full reasoning trace
   printed for you.
4. **Concept B → Task 2.** Read Concept B (the three-question tool-routing demo), then run the
   cell that adds a second tool — a search over your own Day 2 article data — and re-asks a
   policy question and a math question.
5. **Concept C → Task 3.** Read Concept C (the 5-step planning trace), then run the cell with the
   real UPI-limit question — it needs both tools, in an order the agent works out for itself.
6. **Concept D → Tasks 4-5.** Read Concept D (the "when NOT to use one" spectrum and timing
   toggle), then run the cell that times the agent against a plain Python function on a fixed
   task, and the final cell that saves your results.
7. Read the Recap section on the page, then confirm the notebook's final cell has run — it saves
   `m3_1_agent_results.json` next to the notebook.
